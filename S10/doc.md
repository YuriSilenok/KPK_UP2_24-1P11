# Вариант 10: Сервис управления кадрами и отпусками

## Общие требования к API

### Возможные ошибки и коды ответов

| Код | Описание | Когда возникает |
| :--- | :--- | :--- |
| 400 Bad Request | Ошибка валидации входных данных | Неверный формат даты, недопустимое значение перечисления, нарушение ограничений полей |
| 404 Not Found | Сущность не найдена | Запрос по ID несуществующей или удалённой записи |
| 409 Conflict | Конфликт бизнес-логики | Пересечение периодов отпусков/больничных, попытка смены должности при активных отсутствиях, дублирование уникальных полей среди активных записей |
| 500 Internal Server Error | Внутренняя ошибка сервера | Необработанное исключение на стороне сервиса |

### Правила мягкого удаления (Soft Delete)
Для всех сущностей удаление производится путём установки поля `is_active = False`. Метод возвращает `True` при успешном выполнении операции и `False`, если удаление невозможно из-за наличия активных связанных записей (например, для Position при наличии активных EmployeePosition).

## ER-диаграмма

![ER-диаграмма](erd.png)

Диаграмма отображает структуру базы данных сервиса. Сущность Position связана с EmployeePosition отношением «один ко многим». EmployeePosition является центральной сущностью, к которой привязаны Leave и SickLeave через внешние ключи. Profile связан с EmployeePosition для идентификации сотрудника. Все связи обеспечивают целостность данных при мягком удалении через поле is_active.

## Ставка сотрудника (EmployeePosition)

### Добавить ставку

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
