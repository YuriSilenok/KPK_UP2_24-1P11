### Добавить должность

## Ставка сотрудника (EmployeePosition)

### Добавить ставку

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| profile_id | ID профиля сотрудника | Да | Integer | Должен существовать | - |
| position_id | ID должности | Да | Integer | Должен существовать | - |
| start_date | Дата начала ставки | Да | Date | Не может быть в будущем | - |
| end_date | Дата окончания ставки | Нет | Date | Должна быть >= start_date | NULL |
| rate | Коэффициент ставки | Да | Decimal | Диапазон [0.1; 2.0] | - |
| is_primary | Признак основной ставки | Нет | Boolean | Только одна активная основная ставка на сотрудника | false |

**Уникальные комбинации параметров:**
*   У одного сотрудника (`profile_id`) может быть только одна активная основная ставка (`is_primary=true`).

### Изменить ставку по ID

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| position_id | ID новой должности | Нет | Integer | Нельзя менять при активных отпусках/больничных | - |
| start_date | Дата начала ставки | Нет | Date | Не может быть в будущем | - |
| end_date | Дата окончания ставки | Нет | Date | Должна быть >= start_date | - |
| rate | Коэффициент ставки | Нет | Decimal | Диапазон [0.1; 2.0] | - |
| is_primary | Признак основной ставки | Нет | Boolean | Автоматически сбрасывает предыдущую основную ставку | - |

**Ошибки при изменении:**
*   **409 Conflict**: «Невозможно изменить должность: у сотрудника есть активные отпуска или больничные листы». Возвращается при попытке смены `position_id`, если существуют активные периоды отсутствия.

**Ответ при успешном изменении (200 OK)**

| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| id | Идентификатор записи | Integer |
| profile_id | ID профиля сотрудника | Integer |
| position_id | ID должности | Integer |
| start_date | Дата начала ставки | Date |
| end_date | Дата окончания ставки | Date |
| rate | Коэффициент ставки | Decimal |
| is_primary | Признак основной ставки | Boolean |
| is_active | Статус активности | Boolean |

### Удаление ставки по ID

Запись не удаляется физически, а помечается как неактивная через поле `is_active`. Метод возвращает `True` при успехе.

| end_date | Дата окончания ставки | Date |

| is_primary | Признак основной ставки | Boolean |
| is_active | Статус активности | Boolean |

## Отпуск (Leave)

### Добавить отпуск

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| employee_position_id | ID ставки сотрудника | Да | Integer | Должен существовать | - |
| start_date | Дата начала отпуска | Да | Date | - | - |
| end_date | Дата окончания отпуска | Да | Date | Должна быть >= start_date | - |
| leave_type | Тип отпуска | Да | String | Допустимые: ANNUAL, UNPAID, STUDY | - |
| status | Статус отпуска | Нет | String | Допустимые: PLANNED, ACTIVE, COMPLETED, CANCELLED | PLANNED |

**Уникальные комбинации параметров:**
*   Период отпуска не должен пересекаться с другими активными отпусками или больничными на той же ставке (`employee_position_id`).

| leave_type | Тип отпуска | Нет | String | Допустимые: ANNUAL, UNPAID, STUDY | - |
| status | Статус отпуска | Нет | String | Допустимые: PLANNED, ACTIVE, COMPLETED, CANCELLED | - |

## Больничный (SickLeave)

### Добавить больничный

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| employee_position_id | ID ставки сотрудника | Да | Integer | Должен существовать | - |
| start_date | Дата начала больничного | Да | Date | - | - |
| end_date | Дата окончания больничного | Да | Date | Должна быть >= start_date | - |
