## Справочник должностей (Position)

### Создать должность (Create Position)

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| title | Название должности | Да | String | Макс. 255 символов | - |
| code | Код должности | Да | String | Уникальный, макс. 50 символов | - |
| department | Подразделение | Да | String | Макс. 100 символов | - |

**Уникальные комбинации параметров:**
*   Поле `code` должно быть уникальным в рамках всей системы.

| :--- | :--- | :--- |
| id | Идентификатор записи | Integer |
| title | Название должности | String |
| code | Код должности | String |
| department | Подразделение | String |
| is_active | Статус активности | Boolean |
| created_at | Дата создания | DateTime |
| updated_at | Дата последнего обновления | DateTime |

### Получить список должностей (List Positions)

| Параметр фильтрации | Пояснение | Тип | Пример значения |
| :--- | :--- | :--- | :--- |
| department | Фильтр по подразделению | String | "IT" |
| is_active | Статус активности | Boolean | true |

### Обновить должность (Update Position)

## Ставка сотрудника (EmployeePosition)

### Создать ставку (Create EmployeePosition)

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| profile_id | ID профиля сотрудника | Да | Integer | Должен существовать в таблице profiles | - |
| position_id | ID должности | Да | Integer | Должен существовать в таблице positions | - |
| start_date | Дата начала ставки | Да | Date | Не может быть в будущем | - |
| end_date | Дата окончания ставки | Нет | Date | Должна быть после start_date | NULL |
| rate | Коэффициент ставки | Да | Decimal | Диапазон [0.1; 2.0] | - |
| is_primary | Признак основной ставки | Нет | Boolean | Только одна активная основная ставка на сотрудника | false |

**Уникальные комбинации параметров:**
*   У одного сотрудника (`profile_id`) может быть только одна активная основная ставка (`is_primary=true`).

### Получить ставку по ID (Get EmployeePosition by ID)

| Параметр ответа | Пояснение | Тип |
| :--- | :--- | :--- |
| id | Идентификатор записи | Integer |
| profile_id | ID профиля сотрудника | Integer |
| position_id | ID должности | Integer |
| start_date | Дата начала ставки | Date |
| end_date | Дата окончания ставки | Date |
| rate | Коэффициент ставки | Decimal |
| is_primary | Признак основной ставки | Boolean |
| is_active | Статус активности | Boolean |

### Получить список ставок (List EmployeePositions)

| Параметр фильтрации | Пояснение | Тип | Пример значения |
| :--- | :--- | :--- | :--- |
| profile_id | Фильтр по сотруднику | Integer | 101 |
| position_id | Фильтр по должности | Integer | 5 |
| is_active | Статус активности | Boolean | true |
| is_primary | Только основные ставки | Boolean | true |

### Обновить ставку (Update EmployeePosition)

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| position_id | ID новой должности | Нет | Integer | Нельзя менять при активных отпусках/больничных | - |
| start_date | Дата начала ставки | Нет | Date | Не может быть в будущем | - |
| end_date | Дата окончания ставки | Нет | Date | Должна быть после start_date | - |
| rate | Коэффициент ставки | Нет | Decimal | Диапазон [0.1; 2.0] | - |
| is_primary | Признак основной ставки | Нет | Boolean | Автоматически сбрасывает предыдущую основную ставку | - |

**Ответ при успешном обновлении (200 OK)**

| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| id | Идентификатор записи | Integer |
| profile_id | ID профиля сотрудника | Integer |
| position_id | ID должности | Integer |
| start_date | Дата начала ставки | Date |
| end_date | Дата окончания ставки | Date |
| rate | Коэффициент ставки | Decimal |
| is_primary | Признак основной ставки | Boolean |
| is_active | Статус активности | Boolean |

## Отпуск (Leave)

### Создать отпуск (Create Leave)

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| employee_position_id | ID ставки сотрудника | Да | Integer | Должен существовать в таблице employee_positions | - |
| start_date | Дата начала отпуска | Да | Date | - | - |
| end_date | Дата окончания отпуска | Да | Date | Должна быть >= start_date | - |
| leave_type | Тип отпуска | Да | String | Допустимые: ANNUAL, UNPAID, STUDY | - |
| status | Статус отпуска | Да | String | Допустимые: PLANNED, ACTIVE, COMPLETED, CANCELLED | PLANNED |

**Уникальные комбинации параметров:**
*   Период отпуска не должен пересекаться с другими активными отпусками или больничными на той же ставке (`employee_position_id`).

### Получить отпуск по ID (Get Leave by ID)

| :--- | :--- | :--- |
| id | Идентификатор записи | Integer |
| employee_position_id | ID ставки сотрудника | Integer |
| start_date | Дата начала отпуска | Date |
| end_date | Дата окончания отпуска | Date |
| leave_type | Тип отпуска | String |
| status | Статус отпуска | String |
| is_active | Статус активности записи | Boolean |

### Получить список отпусков (List Leaves)

| Параметр фильтрации | Пояснение | Тип | Пример значения |
| :--- | :--- | :--- | :--- |
| employee_position_id | Фильтр по ставке | Integer | 505 |
| status | Фильтр по статусу | String | "ACTIVE" |
| leave_type | Фильтр по типу | String | "ANNUAL" |
| date_from | Начало периода поиска | Date | "2024-01-01" |
| date_to | Конец периода поиска | Date | "2024-12-31" |

## Больничный (SickLeave)

### Создать больничный (Create SickLeave)

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| employee_position_id | ID ставки сотрудника | Да | Integer | Должен существовать в таблице employee_positions | - |
| start_date | Дата начала больничного | Да | Date | - | - |
| end_date | Дата окончания больничного | Да | Date | Должна быть >= start_date | - |
| certificate_number | Номер листка нетрудоспособности | Да | String | Уникальный, макс. 50 символов | - |
| status | Статус больничного | Да | String | Допустимые: OPEN, CLOSED, EXTENDED | OPEN |

**Уникальные комбинации параметров:**
*   Период больничного не должен пересекаться с другими активными больничными или отпусками на той же ставке.

| status | Статус больничного | String |
| is_active | Статус активности записи | Boolean |

### Получить список больничных (List SickLeaves)
