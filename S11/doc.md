# Вариант №11. Сервис дисциплин (Discipline Service)

## 1. Архитектура данных (ER-модель)

Так как одна дисциплина может относиться к нескольким специальностям, а одна специальность включает множество дисциплин, реализуется связь **Many-to-Many**.

### 1. Добавить дисциплину

Создает новую дисциплину и привязывает её к указанным специальностям.

**Входные параметры:**

| Параметр | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- |
| `name` | Да | String | Не пустое | — |
| `code` | Да | String | Уникальное глобально. Формат: `[a-zA-Z0-9.-]` | — |
| `speciality_ids` | Да | Array[Int] | Список существующих ID специальностей. Не пустой. | — |

**Ответ при успехе (201 Created):**

| Параметр | Тип | Описание |
| :--- | :--- | :--- |
| `id` | Integer | ID созданной дисциплины |
| `name` | String | Название |
| `code` | String | Код |
| `speciality_ids` | Array[Int] | Список ID связанных специальностей |

### 2. Изменить дисциплину по ID

Обновляет данные дисциплины. Если передан список специальностей, старые связи удаляются и заменяются новыми.

**Входные параметры:**

| Параметр | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- |
| `name` | Нет | String | Не пустое | Текущее значение |
| `code` | Нет | String | Уникальное глобально. Формат: `[a-zA-Z0-9.-]` | Текущее значение |
| `speciality_ids` | Нет | Array[Int] | Список существующих ID специальностей | Текущие связи сохраняются |

**Ответ при успехе (200 OK):**

| Параметр | Тип | Описание |
| :--- | :--- | :--- |
| `id` | Integer | ID дисциплины |
| `name` | String | Обновленное название |
| `code` | String | Обновленный код |
| `speciality_ids` | Array[Int] | Актуальный список ID специальностей |


### 3. Удалить дисциплину по ID

Удаляет дисциплину и все связанные с ней записи в таблице `DisciplineSpecialty`.

**Параметры:**
*   `id` (Path parameter): Integer

**Ответ:**
*   `True` (Boolean), если удаление прошло успешно.
*   `False` (Boolean), если дисциплина с таким ID не найдена.

### 4. Получить дисциплину по ID

Возвращает полную информацию о дисциплине, включая список связанных специальностей.

**Параметры:**
*   `id` (Path parameter): Integer

**Ответ при успехе (200 OK):**

| Параметр | Тип | Описание |
| :--- | :--- | :--- |
| `id` | Integer | ID дисциплины |
| `name` | String | Название |
| `code` | String | Код |
| `speciality_ids` | Array[Int] | Список ID связанных специальностей |

### 5. Получить список дисциплин (Поиск/Фильтрация)

Возвращает список дисциплин, отфильтрованных по заданным критериям. Все параметры фильтрации необязательны. Логика фильтрации: **AND**.

**Входные параметры (Query parameters):**

| Параметр | Тип | Описание |
| :--- | :--- | :--- |
| `name` | String | Поиск по подстроке в названии (case-insensitive) |
| `code` | String | Поиск по подстроке в коде (case-insensitive) |
| `speciality_id` | Integer | Фильтр по ID специальности (возвращает дисциплины, привязанные к этой специальности) |

**Ответ при успехе (200 OK):**

Список объектов, где каждый объект имеет структуру:

| Параметр | Тип | Описание |
| :--- | :--- | :--- |
| `id` | Integer | ID дисциплины |
| `name` | String | Название |
| `code` | String | Код |
| `speciality_ids` | Array[Int] | Список ID связанных специальностей |
