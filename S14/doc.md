# Load Calculation Service (Сервис расчета нагрузки)

**Вариант 14**

Сервис автоматически рассчитывает, сколько часов должен отработать преподаватель в семестре, на основе данных из других сервисов (учебные планы, количество групп, закрепления).

## Формула расчета

Где:
- `plan_hours` — общее количество часов по учебному плану (из Curriculum Plan Service)
- `groups_count` — количество групп, у которых ведётся дисциплина (из Group Service)

## Сущность: CalculatedLoad (Результат расчёта нагрузки)

Сервис хранит только результат расчёта. Все исходные данные получает из других микросервисов.

| Поле | Тип | Пояснение |
|------|-----|-----------|
| `id` | int | Первичный ключ |
| `teacher_id` | int | ID преподавателя (из Profile/Auth Service) |
| `period_id` | int | ID учебного периода (из Academic Period Service) |
| `total_hours` | float | Рассчитанная общая нагрузка |
| `is_active` | bool | Мягкое удаление |

**Уникальная комбинация:** `teacher_id` + `period_id` (среди активных записей)

---

## 1. Автоматический расчёт и создание

**POST /calculate**

Автоматически запрашивает данные из других сервисов, рассчитывает нагрузку и сохраняет результат.

### Тело запроса

| Параметр | Пояснение | Обязательность | Тип | Ограничение |
|----------|-----------|----------------|-----|-------------|
| `teacher_id` | ID преподавателя | Да | int | > 0 |
| `period_id` | ID учебного периода | Да | int | > 0 |

### Ответ (успешное создание, 201)

| Параметр | Тип |
|----------|-----|
| `id` | int |
| `teacher_id` | int |
| `period_id` | int |
| `total_hours` | float |
| `is_active` | bool |

### Ошибки

| Код | Описание |
|-----|----------|
| 400 | Расчёт для этого преподавателя и периода уже существует |
| 502 | Ошибка при обращении к другому сервису |

---

## 2. Ручное изменение нагрузки

**PUT /loads/{id}**

Для ручной корректировки рассчитанной нагрузки.

### Тело запроса

| Параметр | Пояснение | Обязательность | Тип | Ограничение |
|----------|-----------|----------------|-----|-------------|
| `total_hours` | Новое значение нагрузки | Нет | float | ≥ 0 |

### Ответ

| Параметр | Тип |
|----------|-----|
| `id` | int |
| `teacher_id` | int |
| `period_id` | int |
| `total_hours` | float |
| `is_active` | bool |

---

## 3. Удаление (мягкое удаление)

**DELETE /loads/{id}**

Устанавливает `is_active = false`. Запись физически не удаляется.

| Возвращаемое значение | Тип |
|----------------------|-----|
| `true` | Запись найдена и помечена как удалённая |
| `false` | Запись не найдена или уже удалена |

---

## 4. Получение расчёта по ID

**GET /loads/{id}**

Возвращает только активные записи.

### Ответ

| Параметр | Пояснение | Тип |
|----------|-----------|-----|
| `id` | ID записи | int |
| `teacher_id` | ID преподавателя | int |
| `period_id` | ID учебного периода | int |
| `total_hours` | Общая нагрузка | float |
| `is_active` | Статус активности | bool |

---

## 5. Получение списка расчётов

**GET /loads**

### Параметры запроса

| Параметр | Пояснение | Тип | Обязательность |
|----------|-----------|-----|----------------|
| `teacher_id` | Фильтр по преподавателю | int | Нет |
| `period_id` | Фильтр по периоду | int | Нет |
| `limit` | Максимум записей | int | Нет (по умолчанию 100) |
| `offset` | Пагинация | int | Нет (по умолчанию 0) |

### Ответ

Список объектов `CalculatedLoad` (поля те же, что в п. 4)

---

## 6. Получить всю нагрузку преподавателя

**GET /teachers/{teacher_id}/loads**

Возвращает все активные расчёты нагрузки для указанного преподавателя.

### Ответ

Список объектов `CalculatedLoad`

---

## Зависимости от других сервисов

| Сервис | Назначение |
