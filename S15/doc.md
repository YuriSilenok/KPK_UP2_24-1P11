# 15. Load Assignment Service (Сервис распределения нагрузки)

## Общие коды HTTP-статусов


| Успех | Код | Ошибка | Код |
|-------|-----|--------|-----|
| GET (один) | 200 | Не найдено | 404 |
| GET (список) | 200 | Неверный параметр | 400 |
| POST (создание) | 201 | Конфликт (дубликат) | 409 |
| PUT (обновление) | 200 | Ошибка валидации | 422 |
| DELETE | 200 | Ошибка сервера | 500 |

## Общая структура ошибки


| Параметр | Тип | Пояснение |
|----------|-----|-----------|
| error | string | Тип ошибки |
| message | string | Описание ошибки |

## LoadAssignment
Спецификация сущности распределения нагрузки в базе данных.


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
|:---|:---|:---|:---|:---|:---|
| id | Идентификатор | Да | integer | PK | Нет |
| teacher_id | ID преподавателя | Да | integer | FK -> Teacher.id | Нет |
| discipline_id | ID дисциплины | Да | integer | FK -> Discipline.id | Нет |
| group_id | ID группы | Да | integer | FK -> Group.id | Нет |
| semester | Номер семестра | Да | integer | CHECK (1-8) | Нет |
| load_hours | Часы | Да | decimal(5,2) | CHECK (>0) | Нет |
| is_active | Флаг логического удаления (активности) | Да | boolean | None | True |

## Уникальные комбинации параметров для LoadAssignment
Уникальная комбинация полей: (teacher_id, discipline_id, group_id, semester) — не может существовать двух записей с одинаковыми значениями этих полей.
