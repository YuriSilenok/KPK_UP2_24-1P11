## Спецификация сущности LoadAssignment


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |

### Уникальные комбинации параметров для LoadAssignment
Уникальная комбинация полей: (teacher_id, discipline_id, group_id, semester) — не может существовать двух записей с одинаковыми значениями этих полей.

* **HTTP 422 (Ошибка валидации):** новые значения параметров нарушают ограничения (например, семестр выходит за рамки 1-8).

## Получить список LoadAssignment

### Параметры запроса
> Исправлено по замечаниям №3 и №4: убран синтаксис `CHECK`, перефразированы ограничения для limit/offset, исправлена битая кавычка.


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| teacher_id | Фильтр по преподавателю | Нет | integer | Больше 0 | Нет |
| discipline_id | Фильтр по дисциплине | Нет | integer | Больше 0 | Нет |
| group_id | Фильтр по группе | Нет | integer | Больше 0 | Нет |
| semester | Фильтр по семестру | Нет | integer | От 1 до 8 | Нет |
| limit | Лимит записей | Нет | integer | Целое число, должно быть больше 0 | 100 |
| offset | Сдвиг выборки | Нет | integer | Целое число, не может быть отрицательным | 0 |

### Логика фильтрации параметров
Все переданные параметры запроса (`teacher_id`, `discipline_id`, `group_id`, `semester`) комбинируются по принципу логического **AND**. Поиск выдает только те записи, которые одновременно соответствуют всем указанным фильтрам.

### Информация возвращается в виде списка сущностей (JSON-массив объектов):


| Параметр | Тип |
| :--- | :--- |
| id | integer |
| teacher_id | integer |
| discipline_id | integer |
| group_id | integer |
| semester | integer |
| load_hours | decimal(5,2) |
| is_active | boolean |

### Возможные ошибки для GET (список)
* **HTTP 400 (Неверный параметр):** параметр `limit` меньше или равен 0 или параметр `offset` меньше 0.
