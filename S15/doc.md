# 15. Load Assignment Service (Сервис распределения нагрузки)

## Общие коды HTTP-статусов


| Успех | Код | Ошибка | Код |
|-------|-----|--------|-----|
| GET (один) | 200 | Не найдено | 404 |
| GET (список) | 200 | Неверный параметр | 400 |
| POST (создание) | 201 | Конфликт (дубликат) | 409 |
| PUT (обновление) | 200 | Ошибка валидации | 422 |
| DELETE | 200 | Ошибка сервера | 500 |

| Параметр | Тип | Пояснение |
|----------|-----|-----------|
| error | string | Тип ошибки |

## Спецификация сущности LoadAssignment


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
|:---|:---|:---|:---|:---|:---|
| id | Идентификатор | Да | integer | PK | Нет |
| teacher_id | ID преподавателя | Да | integer | FK -> Teacher.id | Нет |
| discipline_id | ID дисциплины | Да | integer | FK -> Discipline.id | Нет |
| group_id | ID группы | Да | integer | FK -> Group.id | Нет |
| semester | Номер семестра | Да | integer | CHECK (semester BETWEEN 1 AND 8) | Нет |
| load_hours | Часы | Да | decimal(5,2) | CHECK (load_hours > 0) | Нет |
| is_active | Флаг логического удаления (активности) | Да | boolean | None | True |

### Уникальные комбинации параметров для LoadAssignment
Уникальная комбинация полей: (teacher_id, discipline_id, group_id, semester) — не может существовать двух записей с одинаковыми значениями этих полей.

## Добавить LoadAssignment

### Параметры запроса


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
|:---|:---|:---|:---|:---|:---|
| teacher_id | ID преподавателя | Да | integer | CHECK (teacher_id > 0) | Нет |
| discipline_id | ID дисциплины | Да | integer | CHECK (discipline_id > 0) | Нет |
| group_id | ID группы | Да | integer | CHECK (group_id > 0) | Нет |
| semester | Номер семестра | Да | integer | CHECK (semester BETWEEN 1 AND 8) | Нет |
| load_hours | Часы | Да | decimal(5,2) | CHECK (load_hours > 0) | Нет |

### Информация, возвращаемая в случае удачного создания сущности:


| Параметр (значения на английском) | Тип |
|----------|-----|
| id | integer |
| teacher_id | integer |
| discipline_id | integer |
| group_id | integer |
| semester | integer |
| load_hours | decimal(5,2) |
| is_active | boolean |

### Возможные ошибки для POST
* **HTTP 400 (Неверный параметр):** передан некорректный формат данных или тип параметров.
* **HTTP 409 (Конфликт):** нарушение уникальности. Запись с такой комбинацией `teacher_id`, `discipline_id`, `group_id` и `semester` уже существует.
* **HTTP 422 (Ошибка валидации):** параметр `semester` не входит в диапазон 1-8 или `load_hours` <= 0.

## Изменить LoadAssignment по ID

### Параметры запроса


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
|:---|:---|:---|:---|:---|:---|
| id | ID изменяемой записи (передается в Path) | Да | integer | PK | Нет |
| teacher_id | ID преподавателя | Нет | integer | FK -> Teacher.id | Нет |
| discipline_id | ID дисциплины | Нет | integer | FK -> Discipline.id | Нет |
| group_id | ID группы | Нет | integer | FK -> Group.id | Нет |
| semester | Номер семестра | Нет | integer | CHECK (semester BETWEEN 1 AND 8) | Нет |
| load_hours | Часы | Нет | decimal(5,2) | CHECK (load_hours > 0) | Нет |

**Примечание:** Минимум один параметр из необязательных (`teacher_id`, `discipline_id`, `group_id`, `semester`, `load_hours`) обязан быть передан в запросе для выполнения обновления.

### Информация, возвращаемая в случае удачного изменения сущности:


| Параметр (значения на английском) | Тип |
|----------|-----|
| id | integer |
| teacher_id | integer |
| discipline_id | integer |
| group_id | integer |
| semester | integer |
| load_hours | decimal(5,2) |
| is_active | boolean |

### Возможные ошибки для PUT
* **HTTP 400 (Неверный параметр):** не передано ни одного поля для обновления.
* **HTTP 404 (Не найдено):** запись с указанным `id` не существует.
* **HTTP 409 (Конфликт):** в результате обновления создается дубликат уже существующей нагрузки.
* **HTTP 422 (Ошибка валидации):** новые значения параметров нарушают ограничения (например, семестр выходит за рамки 1-8).

## Удалить LoadAssignment по ID

### Параметры запроса


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
|:---|:---|:---|:---|:---|:---|
| id | ID записи (передается в Path) | Да | integer | PK | Нет |

### Возвращаемые данные
Вернёт `True`, если LoadAssignment была закрыта (удалена), иначе вернёт `False`. Фактически запись из БД не удаляется, а реализуется через булевое поле `is_active`.

### Возможные ошибки для DELETE
* **HTTP 404 (Не найдено):** активная запись с указанным `id` не найдена в базе данных.

## Получить список LoadAssignment

### Параметры запроса


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
|:---|:---|:---|:---|:---|:---|
| teacher_id | Фильтр по преподавателю | Нет | integer | CHECK (teacher_id > 0) | Нет |
| discipline_id | Фильтр по дисциплине | Нет | integer | CHECK (discipline_id > 0) | Нет |
| group_id | Фильтр по группе | Нет | integer | CHECK (group_id > 0) | Нет |
| semester | Фильтр по семестру | Нет | integer | CHECK (semester BETWEEN 1 AND 8) | Нет |
| limit | Лимит записей | Нет | integer | CHECK (limit > 0) | 100 |
| offset | Сдвиг выборки | Нет | integer | CHECK (offset >= 0) | 0 |

### Логика фильтрации параметров
Все переданные параметры запроса (`teacher_id`, `discipline_id`, `group_id`, `semester`) комбинируются по принципу логического **AND**. Поиск выдает только те записи, которые одновременно соответствуют всем указанным фильтрам.

### Информация возвращается в виде списка сущностей (JSON-массив объектов):


| Параметр (значения на английском) | Тип |
|----------|-----|
| id | integer |
| teacher_id | integer |
| discipline_id | integer |
| group_id | integer |
| semester | integer |
| load_hours | decimal(5,2) |
| is_active | boolean |

### Возможные ошибки для GET (список)
* **HTTP 400 (Неверный параметр):** параметр `limit` меньше или равен 0, либо параметр `offset` отрицательный.
