# 15. Load Assignment Service (Сервис распределения нагрузки)

| Успех | Код | Ошибка | Код |
|-------|-----|--------|-----|
| GET (один) | 200 | Не найдено | 404 |
| GET (список) | 200 | Неверный параметр | 400 |
| POST (создание) | 201 | Конфликт (дубликат) | 409 |

| DELETE | 200 | Ошибка сервера | 500 |

|----------|-----|-----------|

## Спецификация сущности LoadAssignment


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
|:---|:---|:---|:---|:---|:---|
| id | Идентификатор | Да | integer | PK | Нет |
| teacher_id | ID преподавателя | Да | integer | FK -> Teacher.id | Нет |
| discipline_id | ID дисциплины | Да | integer | FK -> Discipline.id | Нет |
| group_id | ID группы | Да | integer | FK -> Group.id | Нет |
| semester | Номер семестра | Да | integer | CHECK (semester BETWEEN 1 AND 8) | Нет |
| load_hours | Часы | Да | decimal(5,2) | CHECK (load_hours > 0) | Нет |
| is_active | Флаг логического удаления (активности) | Да | boolean | None | True |

### Уникальные комбинации параметров для LoadAssignment
Уникальная комбинация полей: (teacher_id, discipline_id, group_id, semester) — не может существовать двух записей с одинаковыми значениями этих полей.

| Параметр | Тип |

### Возможные ошибки для POST (Замечание №6)
* **HTTP 400 (Неверный параметр):** передан некорректный формат данных или тип параметров.
* **HTTP 409 (Конфликт):** нарушение уникальности. Запись с такой комбинацией `teacher_id`, `discipline_id`, `group_id` и `semester` уже существует.
* **HTTP 422 (Ошибка валидации):** параметр `semester` не входит в диапазон 1-8 или `load_hours` <= 0.

* **HTTP 400 (Неверный параметр):** не передано ни одного поля для обновления.

* **HTTP 422 (Ошибка валидации):** новые значения параметров нарушают ограничения (например, семестр выходит за рамки 1-8).

## Получить список LoadAssignment

### Параметры запроса


| Параметр (значения на английском) | Пояснение (к параметру) | Обязательность | Тип | Ограничение | Значение по умолчанию |
|:---|:---|:---|:---|:---|:---|
| teacher_id | Фильтр по преподавателю | Нет | integer | FK -> Teacher.id | Нет |
| discipline_id | Фильтр по дисциплине | Нет | integer | FK -> Discipline.id | Нет |
| group_id | Фильтр по группе | Нет | integer | FK -> Group.id | Нет |
| semester | Фильтр по семестру | Нет | integer | 1-8 | Нет |
| limit | Лимит записей | Нет | integer | >0 | 100 |
| offset | Сдвиг выборки | Нет | integer | >=0 | 0 |

### Логика фильтрации параметров (Замечание №5)
Все переданные параметры запроса (`teacher_id`, `discipline_id`, `group_id`, `semester`) комбинируются строго по принципу логического **AND**. Поиск выдает только те записи, которые одновременно соответствуют всем указанным фильтрам. Если фильтр не передан, то ограничение по нему не применяется.

### Возвращаемые данные при успехе (HTTP 200) – массив


| Параметр | Тип |
|----------|-----|
| id | integer |
| teacher_id | integer |
| discipline_id | integer |
| group_id | integer |
| semester | integer |
| load_hours | decimal(5,2) |

### Возможные ошибки для GET (список) (Замечание №6)
* **HTTP 400 (Неверный параметр):** параметр `limit` меньше или равен 0, либо параметр `offset` отрицательный.
