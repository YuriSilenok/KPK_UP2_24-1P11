# Отчет по сервису специальностей (Вариант 6)

## ER-диаграмма
![ER-диаграмма](erd.png)

### Описание структуры данных
Архитектура базы данных спроектирована в третьей нормальной форме (3NF):
1. **Сущность SPECIALTY (Специальности):** Содержит явный первичный ключ `id`, уникальный код специальности (`code`), наименование (`name`) и системный флаг активности записи (`is_active`) для реализации механизма мягкого удаления.
2. **Сущность FGOS (ФГОС):** Хранит реестр уникальных кодов стандартов ФГОС (`code`) и первичный ключ `id`.
3. **Сущность SPECIALTY_FGOS (Транзитивная таблица):** Реализует связь типа «многие-ко-многим» между специальностями и ФГОС через внешние ключи `specialty_id` и `fgos_id`. Поля внешних ключей имеют запрет на хранение значений NULL.

## 1. Манипулирование данными: Специальность (Specialty)

### Добавить специальность
**Информация требуемая для создания:**


| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| code | Уникальный код специальности | Да | String | Unique, Not Null | - |
| name | Наименование специальности | Да | String | Not Null | - |

**Перечислить уникальные комбинации параметров:** Поле `code` должно быть уникальным.

**Информация возвращаемая в случае удачного создания:**


| Параметр | Тип |
| :--- | :--- |
| id | Integer |
| code | String |
| name | String |
| is_active | Boolean |

---

### Изменить специальность по ID
**Информация требуемая для изменения по ID:**


| Параметр | Пояснение | Обязательность | Тип | Ограничение |
| :--- | :--- | :--- | :--- | :--- |
| code | Новое уникальное значение кода | Нет | String | Unique, Not Null |
| name | Новое наименование специальности | Нет | String | Not Null |

**Информация возвращаемая в случае удачного изменения:**


| Параметр | Тип |
| :--- | :--- |
| id | Integer |
| code | String |
| name | String |
| is_active | Boolean |

---

### Удаление специальности по ID
Вернет **True**, если специальность была закрыта (удалена), иначе вернет **False**. Фактически запись из БД не удаляется, а реализуется через булевое поле `is_active`.

---

### Получить специальность по ID
**Информация возвращаемая в случае удачного поиска по ID:**


| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| id | Идентификатор записи в базе данных | Integer |
| code | Код специальности | String |
| name | Название специальности | String |
| is_active | Статус активности записи | Boolean |

---

### Получить список специальностей по заданным параметрам
**Информация требуемая для получения списка:**


| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| limit | Ограничение на количество выводимых записей | Integer |
| offset | Смещение относительно начала выборки | Integer |
| search | Строка поиска для фильтрации результатов по коду или названию | String |
| sort_by | Поле для сортировки результатов (`code` или `name`) | String |

**Информация возвращается в виде списка специальностей со следующими столбцами:**


| Параметр | Тип |
| :--- | :--- |
| id | Integer |
| code | String |
| name | String |
| is_active | Boolean |
