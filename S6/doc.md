# Отчет по сервису специальностей (Вариант 6)

## ER-диаграмма
![ER-диаграмма](erd.png)

### Описание структуры данных
Архитектура базы данных спроектирована в третьей нормальной форме (3NF):
1. **Сущность SPECIALTY (Специальности):** Содержит явный первичный ключ `id`, уникальный код специальности (`code`), наименование (`name`) и системный флаг активности записи (`is_active`) для реализации механизма мягкого удаления.
2. **Сущность FGOS (ФГОС):** Хранит реестр уникальных кодов стандартов ФГОС (`code`) и первичный ключ `id`.
3. **Сущность SPECIALTY_FGOS (Транзитивная таблица):** Реализует связь типа «многие-ко-многим» между специальностями и ФГОС через составной первичный ключ, состоящий из внешних ключей `specialty_id` и `fgos_id`. Поля внешних ключей имеют жесткий запрет на хранение значений NULL.

## 1. Манипулирование данными: Специальность (Specialty)

### Добавить специальность
**Информация требуемая для создания:**



| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| id | Идентификатор записи в базе данных | Нет | Integer | Первичный ключ | Генерируется автоматически |
| code | Уникальный код специальности | Да | String | Unique, Not Null | - |
| name | Наименование специальности | Да | String | Not Null | - |

**Перечислить уникальные комбинации параметров:** Поле `code` должно быть уникальным во всей таблице.

**Информация возвращаемая в случае удачного создания:**



| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| id | Идентификатор созданной записи | Integer |
| code | Уникальный код специальности | String |
| name | Название специальности | String |
| is_active | Статус активности записи | Boolean |

---

### Изменить специальность по ID
**Информация требуемая для изменения по ID:**



| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| code | Новый уникальный значение кода | Нет | String | Unique | Изменение не производится, поле не обновляется |
| name | Новое наименование специальности | Нет | String | - | Изменение не производится, поле не обновляется |

**Информация возвращаемая в случае удачного изменения:**



| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| id | Идентификатор записи | Integer |
| code | Актуальный код специальности | String |
| name | Актуальное название специальности | String |
| is_active | Статус активности записи | Boolean |

---

### Удаление специальности по ID
Выполняется методом мягкого удаления. Фактически строка из базы данных не удаляется, а ее системному флагу `is_active` присваивается значение `False`.
* **Входной параметр:** `id` (Integer, Обязательный) — Идентификатор удаляемой специальности.
* **Возвращаемое значение:** 



| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| status | Флаг успешности операции удаления. Принимает значение True, если статус изменен, иначе False | Boolean |

---

### Получить специальность по ID
**Информация возвращаемая в случае удачного поиска по ID:**



| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| id | Идентификатор записи в базе данных | Integer |
| code | Код специальности | String |
| name | Название специальности | String |
| is_active | Статус активности записи (отображает, удалена ли запись логически) | Boolean |

---

### Получить список специальностей по заданным параметрам
**Информация требуемая для получения списка:**



| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| limit | Ограничение на количество выводимых записей (пагинация) | Нет | Integer | Должен быть больше или равен 1 (`ge=1`) | 10 |
| offset | Смещение относительно начала выборки (пагинация) | Нет | Integer | Должен быть больше или равен 0 (`ge=0`) | 0 |
| search | Строка поиска для фильтрации результатов по совпадению в коде или названии | Нет | String | - | Фильтрация не применяется |
| sort_by | Поле, по которому будет отсортирован список | Нет | String | Доступные значения: `code`, `name` | Сортировка не применяется |

**Информация возвращается в виде списка специальностей со следующими столбцами:**



| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| id | Идентификатор записи | Integer |
| code | Код специальности | String |
| name | Название специальности | String |
| is_active | Статус активности записи | Boolean |
