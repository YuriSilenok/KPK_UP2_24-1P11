# Вариант 9. Student Movement Service (Сервис движения студентов):
## 1. Сущность «StudentMovement»

### 1.1. Добавить StudentMovement (POST)

**Информация, требуемая для создания StudentMovement:**

| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
|----------|-----------|----------------|-----|-------------|----------------------|
| student_id | ID студента | Обязательно | Целое число | Должен существовать | - |
| movement_type_id | ID типа движения | Обязательно | Целое число | Должен существовать | - |
| from_group_id | ID исходной группы | Нет | Целое число | Может быть NULL | NULL |
| to_group_id | ID целевой группы | Нет | Целое число | Может быть NULL | NULL |
| movement_date | Дата движения | Обязательно | Дата | - | - |
| reason | Причина движения | Обязательно | Текст | от 1 до 500 символов | - |
| order_number | Номер приказа | Обязательно | Строка | Уникальный | - |
| created_by | Кто создал запись | Обязательно | Строка | от 1 до 100 символов | - |

**Уникальные комбинации параметров:**
- `order_number`

**Информация, возвращаемая в случае удачного создания StudentMovement:**

| Параметр | Тип |
|----------|-----|
| id | Целое число |
| student_id | Целое число |
| movement_type_id | Целое число |
| from_group_id | Целое число |
| to_group_id | Целое число |
| movement_date | Дата |
| reason | Строка |
| order_number | Строка |
| created_by | Строка |
| created_at | Дата |
| updated_at | Дата (null при создании) |
| is_active | Булево значение |

**Примечание:** При создании записи выполняется проверка пересечения интервалов (`check_time_overlap`). Записи со статусом `is_active = false` (cancelled) исключаются из проверки.

---

### 1.2. Изменить StudentMovement по ID (PATCH)

**Информация, требуемая для изменения StudentMovement по ID:**

| Параметр | Пояснение | Обязательность | Тип | Ограничение |
|----------|-----------|----------------|-----|-------------|
| reason | Причина движения | Нет | Текст | от 1 до 500 символов |
| to_group_id | ID группы назначения | Нет | Целое число | Должен существовать |
| movement_date | Дата движения | Нет | Дата | - |

**Примечания:**
- Поле `order_number` не может быть изменено
- При изменении `movement_date` или `to_group_id` выполняется проверка актуальности дат
- Если изменяется только `end_datetime` (применимо к интервалу), проверяется актуальность

**Информация, возвращаемая в случае удачного изменения StudentMovement:**

| Параметр | Тип |
|----------|-----|
| id | Целое число |
| student_id | Целое число |
| movement_type_id | Целое число |
| from_group_id | Целое число |
| to_group_id | Целое число |
| movement_date | Дата |
| reason | Строка |
| order_number | Строка |
| created_by | Строка |
| created_at | Дата |
| updated_at | Дата |
| is_active | Булево значение |

---

### 1.3. Удалить StudentMovement по ID (мягкое удаление)

Вернет `True`, если запись была помечена как удалённая (`is_active = false`), иначе `False`.

Фактически запись из БД не удаляется. Записи со статусом `is_active = false` считаются отменёнными (cancelled) и исключаются из проверок пересечений.

---
